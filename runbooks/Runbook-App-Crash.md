# Runbook: Application Random Failures & Event Viewer Diagnosis

## Scenario
A customer complains that when they click the "Play Movie" button on the Windows Client application, it periodically fails and displays "Video Server Unreachable" or "Unauthorized". They are extremely frustrated and want immediate resolution. 

## Investigation Steps

**1. Windows Client Diagnostics**
- **Action**: Open **Event Viewer** on the Windows machine.
- **Path**: Windows Logs -> Application.
- **Finding**: You found event logs generated by the client application saying: `System.Net.Http.HttpRequestException: The server returned '504 Video Server Unreachable'.`
- **Result**: We confirm the issue is not Windows network stack (Layer 2/3) but an application layer response coming from the server (Layer 7).

**2. Kubernetes Log Analysis (Linux)**
- **Action**: Connect to the Linux server where the cluster is hosted.
- **Command**: `kubectl get pods -l app=netflix-app`
- **Command**: `kubectl logs <netflix-app-pod-name> --tail=50`
- **Finding**: In the raw pod logs, we observe errors: `[ERROR] [Module:app] Video database did not respond while loading movie data.`
- **Result**: The backend Video database connection is dropping periodically.

## Resolution
1. **Immediate Mitigation**: Scaled up the `netflix-app` replication from 2 to 4 pods to handle the connection pooling issue using `kubectl scale deployment netflix-app --replicas=4`.
2. **Root Cause Fix**: Escalated to the development team providing them the exact Kibana log exports and Docker pod names. The development team updated the invalid auth token secrets in Kubernetes using `kubectl create secret`.
3. **Customer Communication**: Emailed the customer verifying that the issue was identified in our central logging system and mitigated, closing the support case within SLA.
